# 消融实验母本故事 Prompt

这份 Prompt 用于在 NovelPilot 的“共创”中制作消融实验母本。它把会阻塞全书设计的基础口径一次性说清，同时把局部创作选择明确交给全书 Loop，以减少没有必要的访谈轮次。建议选择 `参与模式`，完整讨论并批准全书方向后，让 Harness 完成第一故事弧；第二故事弧计划通过人工审批后、尚未开始其中任何章节时，到常驻的“实验室”冻结母本。

全书 Loop 仍会自主判断是否存在真正的高影响矛盾，因此这份 Prompt 不承诺固定讨论轮数；在模型正确吸收全部已确认决定的前提下，通常只需要处理少量实际缺口即可进入审阅。

## 可直接提交给全书 Loop 的 Prompt

```text
我希望共同规划一部现实主义封闭空间群像悬疑小说《退潮前的十一分钟》。以下内容都是我本轮一次性确认的创作决定，请直接吸收进 Book Direction，不要把已经给出答案的事项再次逐项询问。

## 规模、读者体验与叙事方式

- 全书默认 20 章、约 4～5 万字，只采用两个故事弧；如果局部节奏确有需要，Story Arc Loop 可以在总计 18～22 章内微调，不需要为章数再次回到全书讨论。
- 第一故事弧默认 10 章，是所有实验方案共享的完整前文；第二故事弧默认 10 章，是冻结母本后主要比较的续写区间。
- 面向成年悬疑读者，追求克制、可信、可回看的公平推理感，而不是猎奇、强动作或连续死亡刺激。
- 使用第三人称限知群像叙事，主要在唐峤、许澜、陈渡三个视角间轮换。视角人物可以隐瞒、误判或回避，但叙述本身不能向读者说假话。
- 《退潮前的十一分钟》作为默认正式书名；审阅阶段仍可给出更好的候选书名，但无需在讨论阶段专门询问书名。

## 舞台与可验证规则

故事发生在近海的雾岬岛。岛上的旧潮汐观测站即将永久关闭，六名与十二年前实习生程雾死亡事件有关的人，以整理封站档案的名义重新聚集。

- 进出岛屿只有一条潮汐堤道，每次低潮前后约有 70 分钟可以通行；台风增水会让接下来的两个窗口无法使用，使六人至少被隔绝 36 小时。
- 台风登陆后，手机网络、互联网和对外语音通信中断；站内有线电话与本地设备仍可工作，紧急信标只能发出位置和求救状态，不能传递案情。
- 数据室只有一扇门，没有可供人员出入的窗、通风道或秘密通道。进入需要唐峤保管的机械钥匙和顾向潮掌握的密码；门会自动上锁，但可从内部直接离开。
- 数据室门禁记录、数字观测数据和站内时钟使用同一个主时钟；机械潮位记录仪、柴油备用电源记录和潮位井水痕是相互独立的物理时间证据。
- 空间、潮汐、停电、门禁和设备规则一经写入正史就不能为了反转临时修改。

## 固定人物范围

现时核心人物总数就是六人，唐峤包含在六人之内；他邀请的是另外五人。十二年前死亡的程雾是重要的过去人物，但不计入现时六人，也不得在后半程增加新的核心嫌疑人。

1. 唐峤：退休档案员，当年是观测站记录员，本次聚会的召集者；他与林蓁曾是夫妻，旧案后离婚。
2. 顾向潮：退休的前观测站站长，当年拥有最终签字权，也是许澜的前上级。
3. 许澜：当年的潮汐观测员、程雾的直接带教者，如今是沿海防灾工程师。
4. 陈渡：当年的渡船水手、如今的船长，是程雾的表兄，也是最后见过程雾仍能行动的人之一。
5. 林蓁：当年的岛上诊所护士、如今的医生，参与了最初救护，也是唐峤的前妻。
6. 贺闻：一直负责观测站机电维护，熟悉主时钟、备用电源、门禁和机械潮位仪；他与程雾曾长期争论设备误差是否被人为利用。

六人都必须有可理解的现实目标、一个与旧案相关的隐瞒，以及至少一项把推断误当事实的错误认知，但责任不能平均分配，也不能把所有人写成同样有罪。

## 十二年前旧案的真相边界

十二年前，站方为了让一艘载有急救药品并准备转运两名危重病人的临时船赶在台风前通行，擅自延后了危险潮窗，并准备把经过修正的观测数据当作原始记录提交。这个选择有真实的人道收益，也违反安全程序，并可能导致观测站关闭和相关人员失去职业。

程雾发现原始记录与拟提交数据不一致。她后来的死亡必须由以下链条共同造成：一次并非预谋杀人的正面冲突和伤害；数个人出于不同动机分别隐瞒或改动部分事实；至少一人明知仍存在救援机会却延误了行动。缺失的十一分钟与主时钟或观测记录的人为处理有关，同时遮蔽了程雾受伤后仍然活着的时间。

这不是“六人共同开会决定杀人”，也不存在策划一切的唯一幕后黑手。全书 Loop 可以在上述边界内自行确定谁造成最初伤害、谁改动哪项记录、谁延误救援、谁只是在事后沉默；请选取因果最严密、人物代价最有区分度的方案，不需要逐项让我选择。

## 现在发生的事件

封站清点的第一晚，外部电源在 22:13 中断，柴油备用电源十一分钟后恢复。恢复后，数据室仍处于锁闭状态，但十二年前缺失时段对应的半张机械潮位纸出现在室内桌面上，现有门禁记录却显示无人进入；与此同时，唐峤被发现摔伤在潮位井下层平台。他保持清醒，没有失忆，也没有死亡，但拒绝立刻说明自己在那里见到了谁。

当前事件必须在既定空间和设备规则内得到解释，并与旧案形成因果联系；不能只是某人为了逼迫大家认罪而全知全能地布置一切，也不能复制出第二个与旧案完全相同的案件。

## 两个故事弧的明确分工

- 第一故事弧负责建立封闭规则、六人关系、当前事件和相互冲突的旧案叙述。到故事弧结尾，必须通过已经出现的物证推翻“程雾当场死亡”的官方说法，确认她在缺失的十一分钟内仍然活着，并迫使至少一人承认一项会造成现实后果的隐瞒；但不能揭完最初伤害、记录改动和救援延误的完整责任链。第一弧完成后作为实验共享前文。
- 第二故事弧从第一弧已经提交的事实继续，解释当前事件，重建缺失十一分钟的完整顺序，重新解释前文证据，区分法律责任、职业责任和道德责任，并完成真相公开后的关系收束。结局应付出真实代价：至少包含一项法律或职业后果、一段无法恢复的关系，以及一段有限但不廉价的和解。

决定最终真相所需的物证载体必须最迟在第一故事弧中出现；第二故事弧可以发现其新含义、完成交叉验证，但不能在临近结尾时凭空增加决定性证据。

## 写作禁区与授权范围

请避免超自然解释、失忆、人格分裂、双胞胎或替身、突然出现的万能监控、未知密道、临近结尾才登场的幕后主使、无铺垫的秘密亲属，以及依靠角色无理由不说话才能成立的谜题。所有关键反转都必须能够由已经出现的物证、行为、时间限制或人物知识边界回推。

我明确授权全书 Loop 在这些边界内自行确定：六人的具体秘密和错误认知、责任的精确分配、证据出现和误读的顺序、每个故事弧内部的场景安排，以及局部关系变化。这些属于模型应主动完成的创作选择，不是必须由我逐项确认的未决问题。

现在请停留在全书 Loop 的规划阶段，不要直接写章节，也不要生成逐章大纲或提前规划未来更多故事弧。先把以上内容完整合并为可独立阅读的 Book Direction。只有发现会阻止两弧故事成立的真实硬矛盾时，才继续询问当前影响最大的一个问题；不要询问我已经确认的内容，也不要让我在多个同样可行的局部创作方案之间代替你做选择。如果没有这种硬矛盾，请直接标记方向已具备审阅条件。
```

## 冻结操作约定

这份 Prompt 只定义全书层面的创作合同。Story Arc Loop 仍然每次只滚动规划当前故事弧，并给出结构化的建议章节数。

实验母本的目标检查点是：

1. 第一故事弧已经完成，默认 10 章正文和正史补丁均已提交；
2. 第二故事弧已经由 Story Arc Loop 规划，章节数默认确认为 10 章；
3. 参与模式下已经人工批准第二故事弧；
4. 第二故事弧尚未生成或提交任何章节，项目也没有正在运行的 Harness 动作；
5. 在“实验室”选择“冻结为实验母本”。

母本制作只复制允许进入实验的已提交材料，不会推进当前小说。相同项目检查点重复冻结会返回已有母本。
