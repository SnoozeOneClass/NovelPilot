# 消融实验母本故事 Prompt

这份 Prompt 用于在 NovelPilot 的“共创”中制作消融实验母本。建议选择`参与模式`，完整讨论并批准全书方向后，让 Harness 完成第一故事弧；第二故事弧计划通过人工审批后、尚未开始其中任何章节时，到常驻的“实验室”冻结母本。

## 可直接提交给全书 Loop 的 Prompt

```text
我希望共同规划一部约 18～22 章、3.5～5 万字的现实主义封闭空间群像悬疑小说，暂定名《退潮前的十一分钟》；标题可以在讨论中调整。

故事发生在一座受潮汐通道限制的近海孤岛。岛上的旧潮汐观测站即将永久关闭，一名退休档案员以整理封站档案为由，把与十二年前一名实习生死亡事件有关的六个人重新召集到岛上。台风导致交通和通信中断，当年的一段关键记录再次出现：上锁的数据室中，恰好缺失了十一分钟的观测数据。现在的聚会也很快发生异常，使所有人不得不重新面对旧案。

我希望全书采用两个故事弧：

- 第一故事弧约 8～10 章，建立封闭空间规则、人物关系与旧案疑点，并完成一次有实质后果的阶段性揭示；它会作为实验中所有方案共享的前文。
- 第二故事弧约 10～12 章，从第一弧已提交的事实继续调查、反转并收束真相；它会作为后续实验主要比较的生成区间。

核心反转不要落到“唯一幕后黑手策划一切”。十二年前的死亡可以由一次伤害、数个人出于不同动机的隐瞒，以及被延误的救援共同造成；旧案背后还应存在一个在当时具有现实压力、但在道德上仍可争议的选择和集体掩盖。每个人只掌握部分事实，也都可能把自己的推断误当成真相。最终揭示应改变读者对前文行为的理解，但不能靠新人物或最后时刻的新设定完成。

现在请先停留在全书 Loop 的讨论阶段，不要直接写章节，也不要提前写死所有故事弧和逐章剧情。请通过讨论逐步确认并维护：

1. 岛屿空间、潮汐窗口、停电、通信和数据室出入规则，确保封闭条件可计算、可追溯；
2. 十二年前旧案与当前事件的客观时间线，尤其是缺失十一分钟内真正发生的事情；
3. 六名核心人物各自的公开目标、私人动机、秘密、已知事实、错误认知和彼此关系；
4. 全书层面的真相边界、两弧的高层揭示分工，以及哪些信息必须等到后续才可确认；
5. 关键证据如何出现、被误读、被重新解释，所有反转如何在前文留下可回看线索；
6. 结局需要付出的现实代价，以及真相公开后人物关系和责任如何变化。

请避免超自然解释、失忆、人格分裂、突然出现的万能监控、临近结尾才登场的幕后主使，以及依靠角色无理由隐瞒即可成立的谜题。所有关键反转都应能够由已经出现的物证、行为、时间限制或人物知识边界回推。

当这些内容尚有冲突或高影响假设时继续提问；只有方向足以约束后续滚动规划时，再建议我整理并审阅全书方向。
```

## 冻结操作约定

这份 Prompt 只定义全书层面的创作合同。Story Arc Loop 仍然每次只滚动规划当前故事弧，并给出结构化的建议章节数。

实验母本的目标检查点是：

1. 第一故事弧已经完成，约 8～10 章正文和正史补丁均已提交；
2. 第二故事弧已经由 Story Arc Loop 规划，章节数确认在约 10～12 章；
3. 参与模式下已经人工批准第二故事弧；
4. 第二故事弧尚未生成或提交任何章节，项目也没有正在运行的 Harness 动作；
5. 在“实验室”选择“冻结为实验母本”。

母本制作只复制允许进入实验的已提交材料，不会推进当前小说。相同项目检查点重复冻结会返回已有母本。
