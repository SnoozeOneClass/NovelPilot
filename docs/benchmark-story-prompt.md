# 消融实验母本故事 Prompt 1

这份文档记录用于制作第一个消融实验母本的固定起始 Prompt。Prompt 1 模拟的是一名已经形成较完整故事构思、但需要 AI 协助完成系统规划的个人长篇创作者，而不是测试人员编写的工作流指令。

母本生成阶段不属于后续消融实验的被测区间。Prompt 只需要自然地产生一份具有足够人物、时间线、物证和信息边界压力的小说项目；后续 `none`、`full` 和各消融配置之间的可复现性，由冻结后的同一项目检查点保证。

使用时新建小说、选择 `参与模式`，并勾选“创建实验母本项目”；然后把下面代码块中的内容直接提交到“全书共创”。不要把本段说明或文末的冻结步骤一并提交给模型。

## 可直接提交的 Prompt 1

```text
我有一个悬疑小说构思，想请你和我一起把它完善成一部篇幅相对紧凑、可以正式开始创作的长篇小说。

故事暂定名为《退潮前的十一分钟》，但这不一定是最终书名。预计写4～5万字、18～22章，整体分成前后两个阶段。读者主要是喜欢现实主义悬疑和公平推理的成年人。我希望氛围克制、可信，有比较强的回看价值，不依靠连续死亡、强动作或者猎奇情节制造刺激。

故事发生在近海的雾岬岛。岛上有一座即将永久关闭的旧潮汐观测站。六名与十二年前一名实习生死亡事件有关的人，以整理封站档案的名义重新聚集到这里。

进出岛屿只有一条潮汐堤道，每次低潮前后大约有70分钟可以通行。台风带来的增水会让接下来的两个通行窗口消失，使岛上的人至少被隔绝36小时。台风登陆后，手机、互联网和对外语音通信都会中断，但站内有线电话和本地设备仍然可以工作。紧急信标只能发出位置和求救状态，不能用来传递案情。

观测站的数据室只有一扇门，没有能让人出入的窗户、通风道或者秘密通道。进入数据室需要一把机械钥匙和一组密码，两者分别掌握在不同的人手里。门关闭后会自动上锁，但人在里面可以直接离开。

我希望谜题主要建立在潮汐、停电、门禁和观测记录之间可以核对的时间差上。数据室门禁记录、数字观测数据和站内时钟使用同一个主时钟；机械潮位记录仪、柴油备用电源记录和潮位井留下的水痕，则是相互独立的物理时间证据。这些规则一旦确定，就不能为了后面的反转临时修改。

现时的六名核心人物是：

1. 唐峤：退休档案员，当年是观测站记录员，也是这次聚会的召集者。他和林蓁曾经是夫妻，旧案发生后离婚。
2. 顾向潮：退休的前观测站站长，当年拥有最终签字权，也是许澜的前上级。
3. 许澜：当年的潮汐观测员，是死者程雾的直接带教者，如今是沿海防灾工程师。
4. 陈渡：当年的渡船水手，如今是船长。他是程雾的表兄，也是最后见过程雾仍能行动的人之一。
5. 林蓁：当年是岛上诊所的护士，如今是医生，参与过最初的救护，也是唐峤的前妻。
6. 贺闻：一直负责观测站的机电维护，熟悉主时钟、备用电源、门禁和机械潮位仪。他曾经和程雾争论过设备误差是否被人为利用。

现时的核心人物就限定在这六个人，程雾作为过去人物不计算在六人之内。我不希望故事后半段再引入新的核心嫌疑人。六个人都应该有可以理解的现实目标，并且对旧案有所隐瞒、误判或者回避，但他们承担的责任不能平均分配。

十二年前，站方为了让一艘载有急救药品、并准备转运两名危重病人的临时船赶在台风前通过，擅自延后了危险潮窗，还准备把经过修正的观测数据当作原始记录提交。

这个决定确实帮助了船上的病人，但也违反了安全程序。一旦真相公开，观测站可能被关闭，参与者也可能失去工作和职业声誉。

实习生程雾发现了原始记录与拟提交数据之间的差异，随后在观测站死亡。我不希望她的死亡是一次事先策划好的谋杀，而更像是一条由多人共同造成、但责任并不相同的因果链：

有人在正面冲突中造成了并非预谋的伤害；有人出于不同目的隐瞒或修改了部分事实；还有人明知程雾可能仍然活着，却延误了救援。被从记录中抹去的十一分钟，既与主时钟或观测数据的人为处理有关，也遮蔽了程雾受伤后仍然存活的那段时间。

这不是六个人共同商量后决定杀人，也不存在一个预先策划所有事情的唯一幕后黑手。六个人可以分别承担法律责任、职业责任、道德责任或者单纯的沉默责任，不能把他们写成同样有罪。

现在发生的事件是：

封站清点的第一晚，观测站的外部电源在22:13中断，柴油备用电源在十一分钟后恢复。

恢复供电后，数据室仍处于锁闭状态，但十二年前缺失时段对应的半张机械潮位记录纸，却出现在数据室的桌面上。现有门禁记录显示停电前后都没有人进入。

与此同时，唐峤被发现摔伤在潮位井的下层平台。他保持清醒，没有失忆，也没有死亡，但不愿立刻说明自己在那里见到了谁。

现在发生的事情必须能够在前面确定的空间和设备规则内解释，并且和十二年前的旧案存在真实的因果联系。但我不希望它只是某个人为了逼大家认罪而设计的一场全知全能的表演，也不希望它变成十二年前案件的简单重复。

故事前半段主要建立岛上的封闭规则、六个人之间的关系、当前事件，以及他们互相矛盾的旧案叙述。前半段结束时，应该通过已经出现的物证推翻“程雾在冲突后当场死亡”的说法，确认她在被抹去的十一分钟里仍然活着，并迫使至少一个人承认一项会造成现实后果的隐瞒。

但这个阶段不能揭开全部真相，最初的伤害、记录的改动和救援的延误分别由谁造成，应该仍然存在关键缺口。

故事后半段再从已经确认的事实继续，解释现在发生的事件，重建那十一分钟的完整顺序，并重新解释前面出现过的证据。最终需要区分不同人物的法律责任、职业责任和道德责任。

决定最终真相所需的重要物证，最好在故事前半段就已经出现。后半段可以发现它们的新含义或者完成交叉验证，但不要在接近结局时突然增加一件此前完全不存在的决定性证据。

结局需要让人物付出真实代价。我希望至少有人承担法律或职业后果，有一段关系无法恢复，同时也可以保留一段有限但不廉价的和解。

叙事上，我倾向于使用第三人称限知的群像视角，主要在唐峤、许澜和陈渡之间轮换。视角人物可以隐瞒、误判或者回避某些事情，但叙述本身不能故意向读者提供虚假事实。

请避免使用超自然力量、失忆、人格分裂、双胞胎或替身、秘密通道、突然出现的万能监控、没有铺垫的秘密亲属，以及临近结尾才登场的真正幕后主使。关键反转应该能够从已经出现的物证、人物行为、时间限制和人物当时掌握的信息中回推。

我现在还没有完全确定六个人各自隐瞒了什么、谁造成了最初的伤害、谁修改了哪些记录、谁延误了救援，以及证据具体如何出现和被误解。

这些部分希望由你根据人物动机、因果严密程度和悬疑效果，设计出一套整体上最合理的方案，不需要把每一种同样可行的局部选择都交给我决定。

请先和我一起完善这部小说的整体方向。如果我的构思中存在会影响故事成立的关键矛盾或者缺失，再向我确认；其余人物秘密、责任分配、证据顺序和局部关系变化，可以由你选择最合适的设计。
```

## 设计说明

Prompt 1 有意不包含 `Book Direction`、三层 Loop、Harness、审阅阶段、书名确认顺序或实验冻结点等内部术语。这些行为应由 NovelPilot 自己的 Agent 和工作流保证，而不是由用户提前写进 Prompt。

Prompt 中保留了以下压力来源：

- 六名现时人物和一名过去人物之间的多层关系；
- 共享主时钟与三类独立物理时间证据；
- 当前事件与十二年前旧案之间的因果联系；
- 非预谋伤害、记录改动、救援延误和事后沉默之间的不等责任；
- 前后两个阶段的信息披露边界；
- 重要证据必须提前出现的公平推理约束。

具体责任、人物秘密、证据顺序和局部关系变化仍交给全书规划过程完成，使母本既有稳定的可检查约束，也保留 Agent 真正进行创作规划的空间。

## 冻结操作约定

Prompt 1 只用于启动正常的全书共创，不参与后续消融组之间的直接比较。实验母本的目标检查点是：

1. 新建时使用 `参与模式` 并声明“创建实验母本项目”，随后正常完成并批准全书方向；
2. 第一故事弧已经完成，建议约 10 章，正文和正史补丁均已提交；
3. 第二故事弧已经完成规划，建议约 10 章，但尚未生成或提交任何章节；
4. 在参与模式下人工批准第二故事弧；该批准会先停止生成，再自动发布并冻结母本，不需要进入实验室手动冻结；
5. 冻结完成后源项目永久只读，后续从常驻“实验室”查看母本并开始独立的实验流程。

母本制作只复制允许进入实验的已提交材料，不会生成第二故事弧章节或修改既有小说内容。若本地发布失败，第二故事弧的批准仍然保留、源项目也不会继续生成；“实验室”只提供一次可重复的本地发布重试，不会再次调用模型。相同项目检查点重复发布会返回已有母本。
